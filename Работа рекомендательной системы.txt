Примерный алгоритм подсчёта:
Мы будем считать вероятность встречи определнного элемента
после встречи другого
X -> Y, где X, Y - ID(вершины):
пример:
[17, 42, 8, 103] -> 42
[5, 99, 203] -> 17
[203, 17, 88, 42, 7] -> 19
[11, 11] -> 42

17: 42 -> 0,5
    88 -> 0,5
42: 8 -> 0,5
    7 -> 0,5
8: 103 -> 1
103: поскольку 103 стоит в конце и у него нет следующего элемента
    то в качестве вероятностей и чисел возьмём топ 10 ID из всех 
    сессий в качестве вероятности возьмём частоту
    (число встреч / число всех ID)
    В ДАЛЬНЕЙШЕМ ЭТУ ЛОГИКУ НАЗЫВАЮ "ТОП-10"
5: 99 -> 1
99: 203 -> 1
203: 17 -> 1
88: 42 -> 1
7: top10
11: 11 -> 1
----
Эту логику можно рекурсивно усложнить
Для этого мы будем считать вероятность открыть товар через 2 и более раз

пример(17 -> 42 -> 8) 
Мы хотим найти вероятность (17 -> X -> 8)
для этого рассматриваем все известные комбинации 17 -> X и X -> 8
и перемножаем P(17 -> X) и P(X -> 8)

Тогда будет возможно более точно прогнозировать элемент через один

Этот алгоритм не был опробован и является теоретическим. Его эффективность не доказана

Ответ на вопрос на Шаге2:
Случайное разбиение данных не будет подходить в данном случае, по нескольким причинам:
1. Оно подразумевает разбиение вводных данных на два подмножества, что в нашем случае может
    помешать поскольку мы хотим получить обучающую выборку как можно большую, а тестовых
    элементов для проверки работы алгоритма будет достаточно одного элемента сессии
2. Для обучения рекомендательной модели важно учитывать контекст, а это не будет 
    гарантироваться при использовании случайного разбиения, цепочки будут нарушены,
    следовательно в лучшем случае алгоритм будет более глупым, а в худшем абсолюно 
    нерабочим

Хотя на специфичных исходных данных, они могут различаться.
